# 📱 تعليمات بناء وتثبيت التطبيق

## ✅ التغييرات الأخيرة

### 🔔 طلب الأذونات التلقائي
- **الإشعارات**: يتم طلبها تلقائياً عند فتح التطبيق
- **الموقع**: يتم طلبها تلقائياً بعد الإشعارات
- **لا إشعارات وهمية**: تم إزالة جميع الإشعارات الوهمية عند البدء

### 🚫 إصلاح الإشعارات المزعجة
- تم حذف `useOrderNotifications` من صفحة Orders
- لا إشعارات للطلبات القديمة عند فتح التطبيق
- الإشعارات فقط للطلبات الجديدة الحقيقية

## 🛠️ بناء التطبيق

### الطريقة 1: استخدام سكريبت البناء (Linux/Mac)

```bash
chmod +x build-android.sh
./build-android.sh
```

### الطريقة 2: بناء يدوي

```bash
# 1. بناء ملفات الويب
npm run build

# 2. مزامنة Capacitor
npx cap sync android

# 3. بناء APK
cd android
./gradlew clean
./gradlew assembleDebug
```

### الطريقة 3: بناء من Windows

```cmd
rem 1. بناء ملفات الويب
npm run build

rem 2. مزامنة Capacitor
npx cap sync android

rem 3. بناء APK
cd android
gradlew.bat clean
gradlew.bat assembleDebug
```

## 📦 موقع ملف APK

بعد البناء الناجح، ستجد الملف في:

```
android/app/build/outputs/apk/debug/app-debug.apk
```

## 📲 تثبيت APK على الجهاز

### الطريقة 1: عبر USB

```bash
# تأكد من تفعيل USB Debugging على الجهاز
adb install android/app/build/outputs/apk/debug/app-debug.apk
```

### الطريقة 2: نقل ملف APK

1. انسخ ملف `app-debug.apk` إلى جهازك
2. افتح الملف من جهازك
3. اسمح بالتثبيت من مصادر غير معروفة
4. اضغط "تثبيت"

## 🎯 اختبار الأذونات

بعد تثبيت التطبيق:

1. ✅ افتح التطبيق لأول مرة
2. 🔔 يجب أن تظهر نافذة طلب أذونات الإشعارات **تلقائياً**
3. 📍 بعدها مباشرة تظهر نافذة طلب أذونات الموقع **تلقائياً**
4. ✅ لا إشعارات وهمية عند الفتح

## 🔍 التحقق من السجلات

لرؤية سجلات التطبيق والتأكد من عمل الأذونات:

```bash
# عرض جميع السجلات
adb logcat

# فلترة سجلات التطبيق فقط
adb logcat | grep "com.tayardriver.app"

# البحث عن سجلات الأذونات
adb logcat | grep -E "طلب أذونات|permission"
```

ستجد رسائل مثل:
```
🚀 بدء تهيئة التطبيق...
📢 طلب أذونات الإشعارات...
✅ نتيجة أذونات الإشعارات: true
📍 طلب أذونات الموقع...
✅ نتيجة أذونات الموقع: true
✅ تم تهيئة التطبيق بنجاح
```

## ⚠️ ملاحظات مهمة

1. **مزامنة Capacitor**: بعد أي تغيير في الكود يجب تشغيل:
   ```bash
   npm run build
   npx cap sync android
   ```

2. **إلغاء التثبيت القديم**: إذا كنت تقوم بالتحديث، امسح التطبيق القديم أولاً لضمان تطبيق التغييرات

3. **الأذونات**: يمكن إعادة طلب الأذونات من الإعدادات إذا تم رفضها

## 🐛 حل المشاكل

### المشكلة: الأذونات لا تُطلب تلقائياً

**الحل:**
```bash
# تأكد من مزامنة Capacitor
npx cap sync android

# أعد بناء APK
cd android
./gradlew clean
./gradlew assembleDebug
```

### المشكلة: إشعارات وهمية عند الفتح

**الحل:** تم إصلاحها! تأكد من استخدام أحدث نسخة بعد المزامنة

### المشكلة: لا يمكن التثبيت على الجهاز

**الحل:**
1. فعّل "USB Debugging" في إعدادات المطور
2. اسمح بالتثبيت من مصادر غير معروفة
3. تأكد من اتصال USB صحيح

## 📞 دعم إضافي

إذا واجهت أي مشاكل:
1. تحقق من السجلات باستخدام `adb logcat`
2. تأكد من إصدار Java المناسب (Java 11 أو أحدث)
3. تأكد من تثبيت Android SDK بشكل صحيح
