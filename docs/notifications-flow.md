# كيف يعمل نظام الإشعارات 🔔

## الجداول المراقبة

يتم فحص **جدولين** لإرسال الإشعارات:

### 1️⃣ ✅ `captain_requests` (حالة `pending`)

```sql
SELECT * FROM captain_requests
WHERE status = 'pending'
ORDER BY created_at DESC
```

**الغرض**: طلبات الكابتن (رحلات الركاب)
**نوع الإشعار**: "طلب كابتن"
**تكرار الإشعار**: **مرة واحدة فقط** لكل طلب

### 2️⃣ ✅ `driver_waiting_list` (حالة `pending`)

```sql
SELECT * FROM driver_waiting_list
WHERE status = 'pending'
ORDER BY created_at DESC
```

**الغرض**: طلبات المتاجر في قائمة الانتظار
**نوع الإشعار**: "توصيل طلب متجر"
**تكرار الإشعار**: **مرة واحدة فقط** لكل طلب

---

## آلية عمل الإشعارات

### 1️⃣ **الفحص الدوري** (كل 15 ثانية)
- يتم جلب جميع الطلبات من `captain_requests` و `driver_waiting_list` بحالة `pending`
- يتم عرضها في صفحة الطلبات

### 2️⃣ **اكتشاف الطلبات الجديدة**
```typescript
// في useOrderNotifications.ts
const pendingOrders = deliveries.filter(d => d.status === 'pending')
const newOrders = pendingOrders.filter(order =>
  !notifiedOrdersRef.current.has(order.id)
)
```

### 3️⃣ **إرسال إشعار واحد فقط**
- عند اكتشاف طلب جديد، يتم:
  1. إرسال إشعار للسائق
  2. تشغيل صوت التنبيه
  3. **تسجيل معرف الطلب** في `notifiedOrdersRef`
  4. **منع** إرسال إشعار آخر لنفس الطلب

### 4️⃣ **التنظيف التلقائي**
- عندما يتغير حالة الطلب من `pending` إلى `accepted` أو يُحذف
- يتم إزالة معرفه من قائمة الطلبات المُشعَر بها
- هذا يحرر الذاكرة ويمنع التراكم

---

## مثال على سير العمل

### طلب متجر:
```
1. متجر يضيف طلب جديد → driver_waiting_list (status: pending)
                          ↓
2. التطبيق يكتشف الطلب الجديد (كل 15 ثانية)
                          ↓
3. معرف الطلب غير موجود في notifiedOrdersRef
                          ↓
4. إرسال إشعار "توصيل طلب متجر" + صوت + تسجيل المعرف
                          ↓
5. السائق يقبل الطلب → status: accepted
                          ↓
6. لا يتم إرسال إشعار مرة أخرى ✅
```

### طلب كابتن:
```
1. عميل يطلب كابتن → captain_requests (status: pending)
                          ↓
2. التطبيق يكتشف الطلب الجديد (كل 15 ثانية)
                          ↓
3. معرف الطلب غير موجود في notifiedOrdersRef
                          ↓
4. إرسال إشعار "طلب كابتن" + صوت + تسجيل المعرف
                          ↓
5. السائق يقبل الطلب → status: accepted
                          ↓
6. لا يتم إرسال إشعار مرة أخرى ✅
```
